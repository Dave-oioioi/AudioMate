# AudioMate v0.1.1 Release Notes

This patch fixes the first installer build where ordinary app audio did not lower music/BGM.

## Fixed

- Added background render-session polling for ordinary audio ducking.
- Non-music apps above the trigger threshold now raise the `OtherAudio` ducking trigger.
- Music/BGM apps configured through built-in presets or scan confirmation are lowered while other apps are audible.
- The worker recovers quietly if an audio scan fails.

## Changed

- Tray menu, settings window, scan prompts, and notifications are now localized in Chinese.
- Scan prompts now ask whether each unknown audible app should be added as a music/BGM target.

## Verification

```powershell
dotnet test AudioMate.sln --configuration Release
powershell -ExecutionPolicy Bypass -File scripts\publish-release.ps1 -Version 0.1.1
```
